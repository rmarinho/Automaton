What is a Finite Automaton?

A finite automaton (FA) is an abstract machine that recognizes patterns in input strings. It consists of a finite set of states, an alphabet of input symbols, transitions between states, a start state, and a set of accepting states.

Formally: M = (Q, Σ, δ, q₀, F) where Q is the set of states, Σ is the alphabet, δ is the transition function, q₀ is the start state, and F ⊆ Q are accepting states.

Types of Automata

DFA — Deterministic Finite Automaton

Exactly one transition per state per symbol. δ: Q × Σ → Q

For every (state, symbol) pair, there is exactly one next state.

NFA — Non-deterministic Finite Automaton

Multiple transitions allowed per symbol. δ: Q × Σ → P(Q)

A string is accepted if any path through the automaton reaches an accept state.

ε-NFA — NFA with Epsilon Transitions

Like NFA but allows transitions without consuming input (ε-transitions).

δ: Q × (Σ ∪ {ε}) → P(Q). Epsilon closure extends the reachable states.

DSL Format

automaton DFA
alphabet: a,b
states: q0,q1,q2
start: q0
accept: q2
transitions:
  q0,a -> q1
  q0,b -> q0
  q1,b -> q2
  q2,a -> q1

Use eps for epsilon transitions in ε-NFA definitions.

Key Algorithms

Epsilon Closure

Given a set of states, find all states reachable via ε-transitions alone. Computed as a fixed point: start with the initial set, add all states reachable by ε, repeat until no new states are found.

Subset Construction (NFA → DFA)

Convert an NFA to an equivalent DFA. Each DFA state represents a set of NFA states. Start with ε-closure of the NFA start state. For each symbol, compute the union of transitions from all states in the set.

Minimization

Reduce a DFA to its minimum number of states using partition refinement (Hopcroft's algorithm). States are equivalent if they produce the same accept/reject result for all possible input strings.

Automaton Designer

Version 1.0.0

A tool for designing, parsing, simulating, and testing finite automata (DFA, NFA, ε-NFA).

Features

  • Visual canvas with draggable states
  • Custom DSL editor with live parsing
  • Step-by-step simulation with state highlighting
  • Batch test runner
  • NFA→DFA conversion
  • AI-powered learning assistant
  • OAuth login (GitHub, Google, Microsoft, Apple)
